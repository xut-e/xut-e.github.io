<h2>Pentesters</h2>
Padding oracle asiste efectivamente en desencriptar información sensible sin conocer la clave de encriptación. Como pentester, asegúrate de seguir las prácticas para explotar la vulnerabilidad.

- **Black-Box Testing:** Identifica información encriptada en los inputs de la aplicación o las respuestas. Una vez tengas la información encriptada, modifica el ciphertext byte a byte y observa el servidor en busca de patrones de error.
- **Grey-Box Testing:** El pentester tiene acceso parcial al código de la aplicación o a la documentación de la API para analizar cómo se manejan los errores.
- **Usa Automatización:** Usa herramientas como padbuster para automatizar el proceso de identificar respuestas del servidor y modificar los ciphertext o vectores de inicialización para incrementar la eficiencia de la explotación.

<h2>Medidas de Mitigación para Desarrolladores</h2>
Como desarrollador, puedes tomar las siguientes medidas para prevenir este ataque:

- **Usa Encriptación Autentificada:** Asegúrate de usar la encriptación de autentificación como AES-GCM o AES-CCM, la cual combina encriptación y  autentificación para prevenir la manipulación del ciphertext.
- **Evita Revelar Errores:** Evita mostrar errores como padding inválido en producción.
- **Valida Inputs de Forma Segura:** Evita procesar inputs inválidos, como ciphertext del lado del servidor.
- **Mantén las Librerías Actualizadas:** Usa las últimas librerías de criptografía para evitar vulnerabilidades de versiones anticuadas.
