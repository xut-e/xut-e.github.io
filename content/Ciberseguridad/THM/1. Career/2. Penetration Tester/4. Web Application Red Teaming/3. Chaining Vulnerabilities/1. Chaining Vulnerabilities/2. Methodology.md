<h2>Cómo Pensar Como un Atacante</h2>
Un atacante competente, rara vez encuentra una sóla vulnerabilidad y gana inmediatamente, exploran la aplicación como un usuario curiosos, identificando grietas pequeñas y construyendo poco a poco un camino hacia el objetivo. El poder real de encadenar vulnerabilidades viene de entender cómo interaccionan.

Esta sección introduce un proceso repetible para ayudarte a pensar como un atacante. Aprenderás a cómo mapear la aplicación, identificar debilidades potenciales y combinarlas para conseguir un objetivo mayor, como haría un adversario.

<h3>Paso 1: Usa la Aplicación como un Usuario Normal</h3>
Antes de ninguna otra cosa, explora la aplicación sin asumir nada. Registra una cuenta, inicia sesión, prueba, y comprende qué funcionalidades están disponibles. No vayas a cazar bugs todavía. Simplemente mira cómo es el flujo, los roles de los usuarios, y dónde ocurren las acciones sensibles.

<h3>Paso 2: Enumera y Encuentra Vulnerabilidades</h3>
Ahora cambia tu foco a identificar puntos débiles. Esto pueden ser vulnerabilidades clásicas como SQLi, XSS o IDOR.

En este punto lista todo lo que encuentres, incluso si parece de bajo riesgo.

<h3>Paso 3: Entiende lo que Permite cada Hallazgo</h3>
Una vez que tengas un manojo de debilidades, evalúa su aislamiento. Pregúntate:  "¿Qué puedo hacer con esto si asumo que nada más está roto?"

- ¿Corre este XSS en un contexto útil?
- ¿Puede esta respuesta de login ayudarme a enumerar usuarios?
- ¿Puede esta funcionalidad de subida de archivos ayudarme a colar un script en algún lado?

Aquí estás tratando de comprender el potencial individual de cada una de ellas.

<h3>Paso 4: Piensa como un Atacante, ¿Cuál es el Objetivo?</h3>
Ahora pregúntate: "¿Qué querría el Atacante hacer con esta Aplicación?"

- ¿Robar información sensible?
- ¿Acceder al panel de administración?
- ¿Escalar a RCE?

El contexto importa. Una XSS en un blog es molesta, ¿pero en un dashboard de banca? Eso es una amenaza muy diferente.

<h3>Paso 5: Construye un Camino desde las Debilidades al Objetivo</h3>
Aquí es donde conectas los puntos. Mira tu lista de vulnerabilidades e imagina un camino lógico de un usuario externo al objetivo del atacante.

Ejemplo:

`Verbose login -> Valid usernames -> Política de contraseña débil -> Login -> Stored XSS -> Admin visits -> XSS fires -> Admin cookie theft -> Privilege escalation`

La clave es mapear el camino de exploit paso a paso.

<h3>Paso 6: Ejecuta y Valida cada Paso</h3>
Camina por la cadena en orden. No asumas que algo funcionará, compruébalo.

- ¿Puedes hacerle fuerza bruta a una contraseña?
- ¿Tu XSS se dispara en el contexto correcto?
- ¿Te permitirá la cookie robada acceder al dashboard del admin?

Esto también ayuda a descubrir bloqueadores o dependencias. Puede que el XSS sólo funcione en ciertas páginas o que la fuerza bruta falle por límite de frecuencia.

<h3>Paso 7: Reporta la Cadena Completa</h3>
Al reportar, no aísles bugs. Cuenta la historia completa.

Comienza con un atacante no autentificado puede hacer X, y ve avanzando en la cadena. También debes constatar la amenaza e impacto de la cadena completa respecto a las vulnerabilidades individuales.
