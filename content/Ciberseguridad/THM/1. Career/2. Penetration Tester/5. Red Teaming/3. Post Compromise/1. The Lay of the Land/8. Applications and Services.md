<h2>Aplicaciones Instaladas</h2>
Primero empezamos enumerando el sistema en busca de aplicaciones instaladas comprobando el nombre y versión de la aplicación. Puede que encontremos software vulnerable instalado para explotar y escalar nuestros privilegios de sistemas. También puede que encontremos información como credenciales en texto plano.

Usaremos el comando `wmic` para listar las aplicaciones instaladas y sus versiones.

```powershell
PS C:\Users\thm> wmic product get name,version
Name                                                            Version
Microsoft Visual C++ 2019 X64 Minimum Runtime - 14.28.29910     14.28.29910
AWS Tools for Windows                                           3.15.1248
Amazon SSM Agent                                                3.0.529.0
aws-cfn-bootstrap                                               2.0.5
AWS PV Drivers                                                  8.3.4
Microsoft Visual C++ 2019 X64 Additional Runtime - 14.28.29910  14.28.29910
```

Otra cosa interesante es buscar strings específicas, directorios escondidos, archivos de backup, etc. Entonces podemos usar cmdlets de PowerShell como `Get-ChildItem`, así:

```powershell
PS C:\Users\thm> Get-ChildItem -Hidden -Path C:\Users\kkidd\Desktop\
```

<h2>Servicios y Procesos</h2>
Los servicios de Windows habilitan al administrador del sistema a crear aplicaciones ejecutables en nuestras sesiones Windows. A veces, los servicios de Windows tienen malas configuraciones en permisos, lo que escala al usuario normal a otro con más privilegios. Es por esto que debemos mirar los servicios en ejecución.

El descubrimiento de procesos es un paso de enumeración para entender lo que el sistema ofrece. El red team debería obtener detalles ee información sobre servicios y procesos en ejecución en el sistema. Necesitamos entender la mayor cantidad de información sobre nuestros objetivos. Esta información podría ayudarnos a entender software común en ejecución en los sistemas o en la red. Por ejemplo, el sistema comprometido puede tener una aplicación de cliente personalizada para propósitos internos. El software personalizado desarrollado internamente es una de las causas más comunes de escalada de privilegios.
