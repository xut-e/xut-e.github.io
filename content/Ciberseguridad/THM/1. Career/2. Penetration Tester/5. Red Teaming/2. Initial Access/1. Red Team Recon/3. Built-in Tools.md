Esta tarea se enfoca en:

- `whois`
- `dig`, `nslookup`, `host`
- `traceroute`/`tracert`

Antes de empezar a usar `whois`, vamos a ver los que es WHOIS. WHOIS es un protocolo de petición y respuesta que sigue la especificación [RFC 3912](https://www.ietf.org/rfc/rfc3912.txt). Un servidor WHOIS escucha en el puerto TCP 43 peticiones entrantes. El registro de dominio es responsable de mantener los registros WHOIS para los dominios que asigna. La herramienta `whois` le pide al servidor WHOIS los registros guardados. En el siguiente ejemplo puedes ver que `whois` nos ofrece:

- Registrador del servidor WHOIS
- URL del registrador
- Fecha de creación del registro
- Fecha de actualización del registro
- Contacto y dirección de la persona registrada (a no ser que se oculte por privacidad)
- Contacto y dirección del admin (a no ser que se oculte por privacidad)
- Contacto y dirección de la tecnología (a no ser que se oculte por privacidad)

Como podemos ver arriba, es posible ganar mucha información valiosa con sólo un nombre de dominio. Después de mirar `whois`, podemos tener suerte y encontrar nombres, direcciones de correo electrónico, direcciones de portales, números de teléfono, además de información técnica. Al final de la query `whois`, podemos encontrar name servers autoritativos para el dominio en cuestión.

Las queries DNS pueden ser ejecutadas con diferentes herramientas encontradas en nuestros sistemas, especialmente en sistemas de tipo UNIX. Una herramienta común encontrada en sistemas de tipo UNIX o Windows es `nslookup`. En la siguiente query podemos ver que `nslookup` usa el servidor DNS por defecto para obtener los registros A y AAAA relacionados con el dominio.

```bash
pentester@TryHackMe$ nslookup cafe.thmredteam.com
Server:		127.0.0.53

Name:	cafe.thmredteam.com

Name:	cafe.thmredteam.com

Name:	cafe.thmredteam.com

Name:	cafe.thmredteam.com
Address: 2606:4700:3034::6815:5da9
```

Otra herramienta encontrada en sistemas de tipo UNIX es `dig`, siglas de Domain Information Groper. La herramienta `dig` ofrece un montón de opciones de query e incluso nos permite especificar un servidor DNS diferente para usar. Por ejemplo, podemos usar el servidor DNS de Cloudflare: `dig @1.1.1.1 tryhackme.com`.

```bash
pentester@TryHackMe$ dig cafe.thmredteam.com @1.1.1.1

; <<>> DiG 9.16.21-RH <<>> cafe.thmredteam.com @1.1.1.1
;; global options: +cmd
```

La herramienta `host` es otra alternativa útil para obtener registros DNS de servidores DNS. Considera el siguiente ejemplo:

```bash
pentester@TryHackMe$ host cafe.thmredteam.com
cafe.thmredteam.com has address 172.67.212.249
cafe.thmredteam.com has address 104.21.93.169
cafe.thmredteam.com has IPv6 address 2606:4700:3034::ac43:d4f9
cafe.thmredteam.com has IPv6 address 2606:4700:3034::6815:5da9
```

La herramienta final que viene con todos los sistemas de tipo UNIX es `traceroute` o `tracert` en MS Windows. Como indica el nombre, traza la ruta tomada por los paquetes de nuestro sistema al host objetivo. El output de consola debajo muestra que `traceroute` nos ha dado los saltos de routers que nos conectan al sistema objetivo. Algunos routers no responden a los paquetes mandados por `traceroute`, por lo que vemos una fila de `*` en lugar de la dirección IP.

En resumen, siempre podemos confiar en:

- `whois` para hacer queries a la base de datos WHOIS.
- `nslookup`, `dig` o `host` para hacer queries de los servidores DNS.

Las bases de datos WHOIS y los servidores DNS tienen información pública disponible y hacer queries a cualquiera no genera tráfico sospechoso.

Además, podemos confiar en Traceroute para descubrir los saltos entre nuestro sistema y el objetivo.
