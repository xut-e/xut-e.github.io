Existen muchos scripts que realizan enumeración del sistema de manera similar a los que hemos visto en tareas anteriores. Estas herramientas pueden acortar el proceso de enumeración y revelar diferentes vectores potenciales de escalada de privilegios. Aunque debes recordar que puede perderse algunos.

Debajo hay algunas de las herramientas más comúnmente utilizadas.

----------------------------
<h2>WinPEAS</h2>
Es un script desarrollado para enumerar el sistema objetivo para descubrir vectores de escalada de privilegios. Puedes encontrar WinPEAS [aquí](https://github.com/carlospolop/PEASS-ng/tree/master/winPEAS). Es buena idea redirigir el output a un archivo debido a su voluminosidad.

----------------------------
<h2>PrivescCheck</h2>
Es un script de PowerShell que busca vectores de escalada de privilegios en el sistema objetivo. Ofrece una alternativa a WinPEAS sin requerir la ejecución de un binario. Puedes encontrarlo [aquí](https://github.com/itm4n/PrivescCheck). 

>[!NOTE] Puede que para poder ejecutarlo necesites sobrepasar las políticas de ejecución, usa `Set-ExecutionPolicy`.

```powershell
PS C:\> Set-ExecutionPolicy Bypass -Scope process -Force
PS C:\> . .\PrivescCheck.ps1
PS C:\> Invoke-PrivescCheck
```

---------------------------------
<h2>WES-NG: Windows Exploit Suggester - Next Generation</h2>
Algunos scripts de sugerencia de exploits requerirán que los subas al sistema objetivo y los ejecutes. Esto puede hacer que software antivirus lo detecte y elimine. Para evitar hacer ruido innecesario que pueda atraer atención, puede que prefieras usar WES-NG, que correrá en tu máquina.

WES-NG es un script de Python que puedes encontrar [aquí](https://github.com/bitsadmin/wesng).

Una vez instalado, y antes de usarlo, escribe `wes.py --update` para actualizar la base de datos. El script se referirá a la base de datos que crea para comprobar parches faltantes que puedan resultar en vulnerabilidades en el sistema.

Para usar el script debes ejecutar el comando `systeminfo` en el equipo objetivo. No olvides redirigirlo a un archivo. Una vez hecho esto, puedes ejecutar lo siguiente:

```bash
user@kali$ wes.py systeminfo.txt
```

------------------------------------------
<h2>Metasploit</h2>
Si ya tienes una shell Meterpreter en el sistema objetivo, puedes usar el módulo `multi/recon/local_exploit_suggester` para listar vulnerabilidades que pueden afectar al sistema dado.
