<h2>Companion API - CTF Challenge</h2>

<h3>Welcome, Hacker!</h3>
You've been hired as a security consultant to assess **Companion API**, a mobile backend API for a major event management platform. The organizers want to ensure their attendee management system is secure before the big event.

Your mission: Find the vulnerabilities in this API and retrieve the flag.

<h3>The Scenario</h3>
Companion API is a REST API that handles:

- Attendee authentication
- Ticket management for event sessions
- Internal staff tools

The development team claims the API is "production ready." Prove them wrong.

<h3>Getting Started</h3>

<h4>Prerequisites</h4>

- Docker and Docker Compose installed
- A REST client (curl, Postman, Burp Suite, etc.)
- Your hacking skills

<h4>Running the Challenge</h4>

1. Extract the challenge files:
    
    `unzip web-challenge.zip`
    
2. Start the service:
    
    `./START.sh`
    
3. The API is now available at: **[http://127.0.0.1:8080](http://127.0.0.1:8080)**

To stop the challenge:

```bash
docker compose down
```

<h3>Rules of Engagement</h3>

- This is an **API hacking** challenge - focus on the REST endpoints
- The flag format is: `MWC{...}`
- To finalize the challenge there is an endpoint to get the full flag: `/api/v1/finalize`
    
    - POST request with the following data:
    
    `"fragment_1": "<FRAGMENT_2003>",`
    `"fragment_2": "<FRAGMENT_2>",`
    `"fragment_3": "<FRAGMENT_3>",`
    
    - If data is correct you will get the flag.

<h3>Objective</h3>
Find and submit the flag. The flag proves you've successfully identified and exploited all the intended vulnerabilities in the chain.

Good luck, hacker!
